# Visual Content Creation Guidelines

**Source**: Extracted from CLAUDE.md Visual Content Creation section
**Purpose**: Every image must serve a pedagogical purpose -- visuals illuminate concepts, not decorate.

---

## When to Create Visuals

**Create visuals when they**:
- Simplify complex concepts (e.g., RGB color space diagram)
- Show output examples (e.g., result of running a script)
- Demonstrate progression (e.g., before/after transformation)
- Compare alternatives (e.g., different noise functions side-by-side)
- Visualize data flow (e.g., neural network architecture)
- Illustrate step-by-step processes (e.g., animation frames)

**Do NOT create visuals that**:
- Repeat information already clear in text
- Add decorative "fluff" without educational value
- Require more explanation than they provide
- Distract from the core learning objective

---

## Types of Visuals

### 1. Code Output Images (Required for every exercise)
- Generated by Python scripts in the exercise
- Format: PNG (static) or GIF (animated)
- Resolution: 512x512 or similar

### 2. Conceptual Diagrams
- Explain abstract concepts (color spaces, architectures)
- Created using matplotlib or PIL
- Simple, clean, and high-contrast with labels

### 3. Step-by-Step Visualizations
- Show progression through an algorithm
- Multiple images or animated GIF, each step labeled

### 4. Comparison Grids
- Side-by-side comparison of techniques/parameters
- 2x2 or 3x3 grid layout, each cell labeled

### 5. Architecture Diagrams (for neural network modules)
- Show layer structure, data flow, dimensions

---

## Technical Requirements

| Requirement | Value |
|-------------|-------|
| Resolution | Minimum 150 DPI for diagrams, 72 DPI for screenshots |
| File size | <500 KB per image (PNG compression if needed) |
| Format | PNG for static, GIF for animations (<2 MB) |
| Dimensions | 512x512 to 1024x1024 for outputs, flexible for diagrams |
| Color space | sRGB for consistency |

---

## Animated Diagram Standards

For PIL-based animated diagrams (used throughout the project):
- Canvas: 700x380px (project standard)
- FPS: 15
- Quantization: 64-128 colors (target <500 KB per GIF)
- Hold final frame for ~10 extra frames
- Use shared `_diagram_palette.py` for colors/fonts

---

## Design Principles

- **Simplicity**: Remove unnecessary elements (chartjunk)
- **Contrast**: Ensure text/labels are readable
- **Consistency**: Use same color scheme across module
- **Accessibility**: Include alt text, use colorblind-friendly palettes
- **Annotations**: Label axes, add legends, include scale bars where relevant

## Embedding Visuals

### v2 MDX (`src/content/lessons/<slug>.mdx`)

**Single image** — the `<Figure>` component:
```mdx
<Figure
  src="/lessons/1.1.1/rgb_additive_mixing.png"
  alt="RGB additive colour mixing — three overlapping circles producing cyan, magenta, yellow, and white"
  caption="RGB additive colour mixing: overlapping light creates secondary colours."
  num={1}
  role="diagram"
/>
```

The `src` is a `public/`-rooted absolute path. Assets live in `public/lessons/<slug>/` and are copied from v1 by `scripts/copy-v1-assets.mjs`.

### v1 RST (`content/Module_NN_*/X.Y_subtopic/X.Y.Z_exercise/README.rst`)

**Single image**:
```rst
.. figure:: output.png
   :width: 600px
   :align: center
   :alt: RGB color channels showing red, green, and blue separated

   Output showing the three RGB color channels isolated.
```

**Animated GIF**:
```rst
.. figure:: animation.gif
   :width: 400px
   :align: center
   :alt: Animation showing cellular automaton evolution

   Evolution over 100 generations.
```

**Side-by-Side (list-table)**:
```rst
.. list-table::
   :widths: 50 50

   * - .. figure:: before.png
          :width: 100%
          :alt: Original image

          Before transformation

     - .. figure:: after.png
          :width: 100%
          :alt: Transformed image

          After applying filter
```

## Visual Content Checklist

- [ ] Purpose clear: Can state in one sentence why this visual is needed
- [ ] Quality adequate: Resolution >= 150 DPI, file size < 500 KB
- [ ] Accessibility: Descriptive alt text provided
- [ ] Citation: If adapted from source, cited
- [ ] Labels: Axes, legends, and annotations present where needed
- [ ] Caption: Explains what learner should observe
- [ ] Consistency: Style matches other visuals in module
- [ ] Generated: If code output, script exists and executes successfully
